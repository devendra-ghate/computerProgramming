Q1

Rewrite the following code to make it fast and consume minimum amount of memory. Your program should give exactly the same output as the following program. ( 5 marks ) 

#include<iostream>                                                                   
#include<cmath>                                                                              
int                             
main(){                                                                                                                                                                   
  float a[100][100], ans=0.0;                                                        
                                                                                         for(int i=0; i < 100; i++){             
    for(int j=0; j < 100; j++){                                                      
      a[i][j] = (i+1.0)/(j+1.0);                                                     
    }                                                                                
  }                                                                                  
                                                                                     
  for(int i=0; i < 100; i++){                                                         
    for(int j=99; j >= 0; j--){                                                      
      if(i>=j && i<=j){                                                              
        a[i][j] = a[i][j]*sin(M_PI*(i+2)/2/(j+1))/100;                                         ans += a[i][j];                                                              
      }                                   
    }                                                                                
  }                                                                                                                                                                       
  std::cout << ans << std::endl;                                                     
  return(0);                                                                         
}

Solution : We note:

  1. The second loop is only working on the diagonal elements. So essentially we need only a single for loop for calculation. Only the diagonal entries are of interest. (1 mark)

  2. Now since only the diagonal entries are of interest, we also notice that the first loop is initializing the elements of a in such a way that all the diagonal elements come out to be 1. So this initialization step can also be eliminated. ( 0.5 marks )

  3. We also notice that the division by 100 is being performed repeated and is independent of i and j. Hence it can be moved out of the loop there by saving us many division operations. ( 1 mark )

  4. Similarly, I can define a variable pibytwo and avoid a lot of divisions. Now, you might feel that one more variable is being added and memory consumption has increased. But if we consider the trade off, one extra variable is nothing compared to 100 division operations. So this is a no brainer. ( 0.5 marks )

  5. Also, pibytwo∗(i+2)/(i+1)=pibytwo+pibytwo/(i+1). So I have converted (2 additions + 1 division + 1 multiplication) into (1 addition + 1 division). ( 0.5 marks )

  6. But now I notice that I don't even have to do that addition i+1. I can simply modify the for loop from 0-99 to 1-100. This is a standard trick in computing and you should always keep this in mind while writing any code.(1 mark )

  7. Clearly, sin⁡(θ+π/2)=cos⁡(θ) and hence that addition also goes. ( 0.5 marks )

  8. In fact, π/2 is available in cmath as M_PI_2. You may not know this. So I have not deducted marks for this. We could have saved on one more division operation.

Here are my two versions of the fastest code:

Without M_PI_2

#include<iostream>
#include<cmath>

int
main(){
  float ans=0.0, pibytwo;
  pibytwo = M_PI/2;
  for(int i=1; i <= 100; i++){
    ans +=  cos(pibytwo/(i));
  }
  std::cout << ans/100 << std::endl;
  return(0);
}

With M_PI_2

#include<iostream>
#include<cmath>

int
main(){
  float ans=0.0;
  for(int i=1; i <= 100; i++){
    ans +=  cos(M_PI_2/(i));
  }
  std::cout << ans/100 << std::endl;
  return(0);
}

Both solutions get full marks.

Q3

What is the approx output of the following program. Your answer should also explain the reasoning behind your expected output. Correct answer without reasoning will fetch only a single mark. ( 1 + 2 marks )

#include<iostream>

int main(){
  double eps=1.0;
  while(1 == 1){
    eps = eps/2.0;
    if(1 + eps == 1){
      break;
    }
  }
  std::cout << eps << std::endl;
  return(0);
}

Solution:

This algorithm is essentially trying to find the next number on the floating point number system after 1. The algorithm starts from the value of 1 for eps and successively halves it. The if condition will be satisfied only when the value of eps becomes smaller than the distance between 1 and the next floating point number. This is called as the machine precision. This is a standard algorithm to find out the machine precision of a computer. For a double variable, this value is 2−52≈10−16. Similarly, if I declare eps as a float, then I would get the answer to be 2−32≈10−8.

Q4

Write a program that does following things:

  1. Takes user input of a 1000 digit number and stores it in a character array of minimum required size.

  2. Prints every even decimal place in the reverse order.

For example, for a six digit number 123456, the output should be 642. You program only has to work for numbers with 1000 digits. A general program for numbers wit h any number of digits is not expected.

You cannot use for loop. ( 3 marks )

Note: Such a large number (1000 digits) has been specified to discourage students from hard coding cout statements from all even decimal places.

Solution:

This is a simple task and can be achieved in multiple ways. One of the solutions is given here.

#include<iostream>
#include<cstring>
#define N 1000 
using namespace std;

int main(){
  char nm[N];
  cin >> nm;

  int i=N-1;
  while(i>=0){
    if((i+1)%2 == 0){
      cout << nm[i];
    }
    i--;
  }
  cout << endl;
  return(0);
}

Q5

Write a program that performs following steps.

  1. Declare a double pointer double **M and allocate memory to store a 2×2 matrix.

  2. Take this array as input (column major form) and store it in an array M of datatype double. For e.g. the input

1
2
3
4

corresponds to matrix

[1324]

  1. Write a function det() that takes this array as the input and calculates its determinant and passes it to the main() function.

  2. The value of the determinant is printed from the main() function. ( 2 marks )

Note: You cannot declare the array as a global variable. The array M should be declared in the scope of main() function.

Solution:

#include<iostream>

using namespace std;

double det(double **M){
  return(M[0][0]*M[1][1] - M[1][0]*M[0][1]);
}

int
main(){
  double **M;
  M = new double *[2];
  for(int i=0; i<2; i++){
    M[i] = new double [2];
  }
  for(int i=0; i<2; i++){
    for(int j=0; j<2; j++){
      cin >> M[j][i];
    }
  }
  cout << det(M) << endl;
  return(0);
}
