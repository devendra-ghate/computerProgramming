Now, looking the measurements, it is difficult to tell, what is the governing law that can relate the velocity to time. In other words, we donâ€™t know \(v(t)\). We also note that the velocity measurements are not accurate. Every measurement instrument has some least count and also there are random errors. Hence, at a givem time instant, the velocity of the object is not exactly equal to the velocity measured at that instant, but is in the neighbourhood of the reported velocity.

Such situations are very common in engineering. Least squares method1 is the most common way to find the closest approximation to \(v(t)\) in some sense. However, first we have to assume the functional form of the function. Let us assume a linear form i.e. \(v = a t + b\), where \(a\) and \(b\) can be found using the formula at the end of the question. Once \(a\) and \(b\) are known, we have an estimate of \(v(t)\). We can given any \(t_0\) and get the corresponding \(v_0\). The estimate will be accurate while \(t_0\) is close to the range of \(t\) in which measurements are performed.

Store the time data in the variable t (array of size 11), velocity data in the variable v (array of size 11) and the number of data points in the variable \(n\). \(t\) and \(v\) data should be initialized in the program during the declaration of variable.

Method: These \(n\) data points can be modelled using the equation \(v = at + b\). Your program should find the coefficients \(a\) and \(b\) using the equations given below

\[ a = \frac{ n \sum_{i=1}^{n} t_i v_i - (\sum_{i=1}^{n} t_i )(\sum_{i=1}^{n} v_i)}{n\sum_{i=1}^{n} t_i^2 - (\sum_{i=1}^{n} t_i)^2} , b = \frac{1}{n} \left( \sum_{i=1}^{n} v_i - a \sum_{i=1}^{n} t_i \right) . \]

Write a program for which input is time (\(t\)) and outputs velocity at that time.
