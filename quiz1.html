Note: Your answer should not be more than 5 sentences long.

Solution

  1. Low level languages are fast. High level are slow.
  2. Difficult to write programs in low level languages as compared to high level languages.
  3. Compiled executables from level level language code are OS specific. Interpreted language code can run on any OS.
  4. Low level language code needs to compilted. Interpreted language code is interpreted and executed on-the-fly.

B. Whenever a new CPU comes in the market, the manufacturer (like Intel or AMD) specifies clock speed (in GHz) and the cache size (in MB). Write a short note on the importance of memory cache size on the performance of a CPU. ( 3 marks )

Note: No more than seven sentences. Choose your words carefully.

Solution

In the increasing order of speed, a computer has various types of memories like hard disk, RAM and CPU Cache. When a program works on a data, the data is fetched by the OS from these various memory locations. Accessing data from cache is much faster than accessing it from RAM or hard disk. Hence, a CPU with larger cache size will be able to store larger portion of required data in the cache and will waste lesser number of precious clock cycles in fetching the data. So the overall number of floating point operations per second (FLOPS) will increase.

There are three types of cache: L1, L2 and L3. These are in decreasing order of speed of access. Cache size cannot increased beyond a certain limit as the chip design becomes increasingly complex. Processor with larger cache sizes become increasingly expensive.

Similar effect is noticed for machines with higher RAM size. OS anticipates the user behaviour and loads frequently used programs in the RAM. So when the user calls the program, it opens quickly. However, this consumes large RAM space. So OS is decides what to load based on the available RAM.

Q2

A. The distance between two consecutive floating point numbers is constant. Is this true or false. Answer without explanation will not be graded. (2 marks)

Solution:

False. Distance between two consecutive floating point numbers is distance between the mantissa multiplied by the exponent. Hence, as move away from zero on either side of the real line, the distance between two consecutive floating point increases. For e.g. for double, the distance between two consecutive numbers in the vicinity of zero is \(1\times10^{-16}\) while away from zero it will keep on increasing. For example, for \(x_1 = 1.23456789123456\times10^{200}\), the next number is \(x_2 = 1.23456789123457\times10^{200}\). So the distance between consecutive points is\(\Delta x = 1\times10^{200-16}\) which is a huge number.

A. We want to write a program that will calculate \(y(x=2.1)\), where

\[ y = x^3 + \sin{\frac{x^2}{2} + 2} + \log{3x^2 + 12} + 4x \]

When I say calculate, that means it has to output (cout) the value of \(y\) for \(x=2.1\). Write a memory efficient program to calculate this. A memory efficient program is a program that consumes the least amount of RAM during execution.

Here is a skeleton of the program to get you started.

#include <iostream>
#include <cmath>

using namespace std;

int main(){
   double x = 2.1;
   .
   .
   .
   return 0;
}

The cmath function to calculate log is log(). Hopefully, you will write a well indented code. But it is not essential. ( 4 marks )

Solution

#include <iostream>
#include <cmath>

using namespace std;

int main(){
   double x = 2.1;
   cout << x^3 +sin(x*x/2 + 2) + log(3*x*x + 12) + 4*x << endl;
   return 0;
}

This program will allocate only one double variable. That means it consumes only 8 bytes.

Even more memory efficient program:

#include <iostream>
#include <cmath>

using namespace std;

#define x 2.1;

int main(){
   cout << x^3 +sin(x*x/2 + 2) + log(3*x*x + 12) + 4*x << endl;
   return 0;
}

B. Write the fastest program for the problem in the earlier question. That means, this time we do not worry about how much RAM our program uses but are only concerned about the time it will take to run. (4 marks)

#include <iostream>
#include <cmath>

using namespace std;

int main(){
   double x = 2.1;
   double z;
   z = x*x + 4;
   cout << x*z + sin(z/2) + log(3*z) << endl;
   return 0;
}

I observe that \(x^2 + 4\) occures repeatedly in the expression for \(y\). In the previous memory efficient program, we were calculating \(x^2 + 4\) repeatedly. So I can calculate it once and store it. sin and log and the addition of terms cannot be helped. These operations are same for the memory efficient program as well as the currect program. Apart these operations, the current program performs three multiplications, one division and one addition. If we did not store the z variable then we will perform one power (or 2 multiplications for \(x^3\)), 4 multiplications, and 2 additions.

If we try to measure the speeds of these programs, we will not see any difference because the programs are small. However, imagine calculating this function a million times over and over. Then we will see the difference.

Note 1: For the simplified analysis considered here, we have assumed that allocation, initialisation and accessing of variable z happens in much shorter span than the algebraic operations. As the expression for \(y\) becomes increasingly complex, this holds true.

Note 2: To measure the CPU time taken by a program, you can use clock() function from <ctime> header file. An example can be found here. A more detailed explanation can be found here.
